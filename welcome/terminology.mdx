Consistency improves searchability, onboarding speed, and future automation.

## Capitalization Rules

- Proper Nouns: Loom, Fray, Keystone Pattern, Consent Pulse
- Generic categories: pattern anomaly, status effect (unless in heading)
- Elements & Threads: Capitalize when referenced as types (Arc Pattern, Fury Thread)

## Singular vs Plural

Use singular for entity page titles (Pattern, StatusEffect, Technique). Use plural in narrative lists.

## Pattern Naming

Format: `<Element>/<Emotion>` (e.g., Flow/Serenity). Dual patterns: `<Element1>/<Emotion1>, <Element2>/<Emotion2>`.

## File & Slug Conventions

- Lowercase with hyphens: `fray-escalation`, `dual-braided`
- Index pages should be `index.mdx` when representing a directory root.
