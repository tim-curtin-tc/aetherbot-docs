# AetherbotCard (Core Domain Entity)

## 1. Canonical Definition

An **AetherbotCard** is a persistent collectible (and optionally consumable or licensable) blueprint artifact that defines baseline capability, flavor metadata, rarity classification, and instantiation permissions for derived Aetherbots. It may also function as a gating license for equipping certain techniques or pattern modules.

## 2. Primary Responsibilities

| Responsibility            | Description                                                                      |
| ------------------------- | -------------------------------------------------------------------------------- |
| Blueprint Definition      | Captures archetype stats & permitted capability envelope.                        |
| Rarity & Classification   | Supplies tier, scarcity, and distribution mechanics.                             |
| Ownership & Transfer      | Behaves as tradable asset (if economy enabled).                                  |
| Instantiation License     | Authorizes creation of new Aetherbot instances (single-use or multi-use policy). |
| Capability Unlock Surface | Grants access to techniques/pattern modules otherwise restricted.                |

## 3. Identity & Referencing

| Aspect            | Rule                                                               |
| ----------------- | ------------------------------------------------------------------ |

## 6. Lifecycle & Status

- `ACTIVE`: Distributable and usable for instantiation.
- `VAULTED`: Frozen supply; existing ownership unaffected.
- `RETIRED`: Permanently locked; no new instantiations.

## 7. Invariants & Constraints

| Constraint                                         | Rationale               | Enforcement            |
| -------------------------------------------------- | ----------------------- | ---------------------- |
| code & slug immutability                           | Prevent reference drift | DB & app guard         |
| mintedCount `<=` supplyCap (if cap set)            | Supply integrity        | Transactional check    |
| licensePolicy.remainingUses decremented atomically | Prevent double mint     | Atomic update w/ guard |
| No instantiation if status not ACTIVE              | Economy integrity       | Service validator      |
