# Aetherbot (Core Domain Entity)

## 1. Canonical Definition

An **Aetherbot** is a persistent autonomous (or semi-autonomous) agent entity representing a configurable combat / interaction unit. It binds together capability modules (techniques, pattern adapters), progression attributes, loadout state, and operational constraints. Ownership vs operational stewardship may diverge (Account owns, Human operates).

## 2. Primary Responsibilities

| Responsibility          | Description                                           |
| ----------------------- | ----------------------------------------------------- |
| Capability Composition  | Aggregates equipped techniques / pattern modules.     |
| Progression Tracking    | Maintains rank, experience, specialization vectors.   |
| Operational Constraints | Tracks fatigue, cooldown states, repair requirements. |
| Stewardship Interface   | Exposes assignment to a Human operator.               |
| Integrity Surface       | Monitors anomalous performance / exploit flags.       |

## 3. Identity & Referencing

| Aspect            | Rule                                            |
| ----------------- | ----------------------------------------------- |
| Global Identifier | `aetherbotId`.                                  |
| Ownership         | `ownerAccountId` required.                      |
| Operator          | Optional `operatorHumanId` (must be ACTIVE).    |
| Status            | `ACTIVE`, `DISABLED`, `RETIRED`, `QUARANTINED`. |
| Visibility        | `PUBLIC`, `PRIVATE`, `UNLISTED`.                |

## 6. Lifecycle & Status

- `ACTIVE`: Eligible for events & loadout changes.
- `DISABLED`: Temporarily unusable (repair, policy violation under review).
- `QUARANTINED`: Isolated due to exploit suspicion; cannot participate.
- `RETIRED`: Permanently archived; immutable progression thereafter.

## 7. Invariants & Constraints
